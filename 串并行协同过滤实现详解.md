## 一、项目概述

### 1.1 项目背景

本项目是一个基于**协同过滤**的电影推荐系统，采用**ALS（交替最小二乘法）**算法实现个性化推荐。项目包含两个版本：

| 版本 | 技术栈 | 特点 |
|------|--------|------|
| **串行版本** | Scala + JBLAS | 单线程执行，用于算法验证和性能对比 |
| **并行版本** | Spark MLlib | 分布式计算，支持大规模数据处理 |

## 二、协同过滤算法原理

### 2.1 ALS算法简介

**ALS（Alternating Least Squares，交替最小二乘法）**是一种基于矩阵分解的协同过滤算法。

#### 核心思想

将用户-物品评分矩阵 **R**（m个用户 × n部电影）分解为两个低秩矩阵的乘积：

```
R_{m×n} ≈ U_{m×k} × V_{k×n}^T
```

其中：
- **U**：用户因子矩阵，每行 `U_i` 是用户 i 的 k 维隐因子向量
- **V**：物品因子矩阵，每行 `V_j` 是电影 j 的 k 维隐因子向量
- **k**：隐因子维度（本项目使用 rank=50）

#### 目标函数

```
min_{U,V} Σ_{(i,j)∈R} (R_ij - U_i · V_j^T)² + λ(‖U_i‖² + ‖V_j‖²)
```

- 第一项：重建误差，让预测评分接近真实评分
- 第二项：L2正则化，防止过拟合
- λ：正则化系数（本项目使用 0.01）

#### 交替优化策略

ALS 的核心是"交替"固定一个矩阵，求解另一个矩阵：

**步骤1：固定 V，求解 U**

```
U_i = (V_{Ri} · V_{Ri}^T + λI)^{-1} · V_{Ri} · R_i
```

其中：
- `V_{Ri}`：用户 i 评分过的物品的因子矩阵
- `R_i`：用户 i 的评分向量

**步骤2：固定 U，求解 V**

```
V_j = (U_{Rj} · U_{Rj}^T + λI)^{-1} · U_{Rj} · R_j
```

其中：
- `U_{Rj}`：评分过物品 j 的用户的因子矩阵
- `R_j`：物品 j 的评分向量

**步骤3：反复迭代**直到收敛或达到最大迭代次数。

### 2.2 推荐生成

训练得到用户因子矩阵 U 和物品因子矩阵 V 后，预测用户 u 对电影 m 的评分：

```
R̂(u,m) = U_u · V_m^T
```

即：用户因子向量与物品因子向量的**点积**。

为用户生成 Top-N 推荐：
1. 计算该用户与所有电影的预测评分
2. 按预测评分降序排序
3. 取前 N 部电影作为推荐

```scala
def splitData(ratings: List[Rating]): (List[Rating], List[Rating], List[Rating]) = {
  val shuffled = scala.util.Random.shuffle(ratings)  // 随机打乱
  val total = shuffled.size
  val trainSize = (total * 0.6).toInt
  val valSize = (total * 0.2).toInt
  
  val training   = shuffled.take(trainSize)                    // 前60%
  val validation = shuffled.slice(trainSize, trainSize + valSize) // 中间20%
  val test       = shuffled.drop(trainSize + valSize)           // 后20%
  
  (training, validation, test)
}
```

### 3.6 串行ALS训练（核心算法）

```scala
def trainALS(
  ratings: List[Rating],
  userIndex: Map[Int, Int],
  movieIndex: Map[Int, Int],
  numUsers: Int,
  numMovies: Int,
  rank: Int,           // 隐因子维度（默认50）
  lambda: Double,      // 正则化系数（默认0.01）
  numIterations: Int   // 迭代次数（默认10）
): (DoubleMatrix, DoubleMatrix) = {
  
  // ① 初始化因子矩阵（随机初始化）
  var userFactors = DoubleMatrix.rand(numUsers, rank)   // U: m×k
  var itemFactors = DoubleMatrix.rand(numMovies, rank)  // V: n×k
  
  // ② 构建评分矩阵（稠密表示）
  val ratingMatrix = Array.ofDim[Double](numUsers, numMovies)
  ratings.foreach { r =>
    val u = userIndex(r.user)
    val m = movieIndex(r.product)
    ratingMatrix(u)(m) = r.rating
  }
  
  // ③ 串行迭代训练
  for (iter <- 1 to numIterations) {
    println(s"  Iteration $iter/$numIterations")
    
    // ─── 固定V，更新U（逐个用户串行更新）───
    for (u <- 0 until numUsers) {
      // 找到该用户评分过的所有电影
      val ratedMovies = ratings.filter(_.user == users(userIndex, u))
                               .map(r => movieIndex(r.product))
      if (ratedMovies.nonEmpty) {
        // 构建该用户的评分向量 Ru 和对应物品因子矩阵 Mi
        val Ru = DoubleMatrix.zeros(1, ratedMovies.size)
        val Mi = DoubleMatrix.zeros(rank, ratedMovies.size)
        
        ratedMovies.zipWithIndex.foreach { case (m, idx) =>
          Ru.put(0, idx, ratingMatrix(u)(m))        // Ru: 该用户的评分
          Mi.putColumn(idx, itemFactors.getRow(m))   // Mi: 对应物品的因子向量
        }
        
        // 求解最小二乘: (Mi * Mi' + λI) * Uu = Mi * Ru'
        val A = Mi.mmul(Mi.transpose()).add(DoubleMatrix.eye(rank).mul(lambda))
        val B = Mi.mmul(Ru.transpose())
        val Uu = Solve.solve(A, B)   // JBLAS求解线性方程组
        
        userFactors.putRow(u, Uu.transpose())
      }
    }
    
    // ─── 固定U，更新V（逐个物品串行更新）───
    for (m <- 0 until numMovies) {
      // 找到评分过该电影的所有用户
      val ratedUsers = ratings.filter(_.product == movies(movieIndex, m))
                              .map(r => userIndex(r.user))
      if (ratedUsers.nonEmpty) {
        val Rm = DoubleMatrix.zeros(1, ratedUsers.size)
        val Ui = DoubleMatrix.zeros(rank, ratedUsers.size)
        
        ratedUsers.zipWithIndex.foreach { case (u, idx) =>
          Rm.put(0, idx, ratingMatrix(u)(m))
          Ui.putColumn(idx, userFactors.getRow(u))
        }
        
        // 求解: (Ui * Ui' + λI) * Vm = Ui * Rm'
        val A = Ui.mmul(Ui.transpose()).add(DoubleMatrix.eye(rank).mul(lambda))
        val B = Ui.mmul(Rm.transpose())
        val Im = Solve.solve(A, B)
        
        itemFactors.putRow(m, Im.transpose())
      }
    }
  }
  
  (userFactors, itemFactors)
}
```

**串行瓶颈分析**：

| 操作 | 串行方式 | 时间复杂度 |
|------|---------|-----------|
| 查找用户评分过的电影 | `ratings.filter(...)` 全表扫描 | O(\|ratings\|) 每个用户 |
| 查找评分过电影的用户 | `ratings.filter(...)` 全表扫描 | O(\|ratings\|) 每个电影 |
| 更新用户因子 | `for (u <- 0 until numUsers)` 逐个 | O(numUsers × rank²) |
| 更新物品因子 | `for (m <- 0 until numMovies)` 逐个 | O(numMovies × rank²) |
| 单次迭代总计 | 顺序执行 | O((numUsers + numMovies) × \|ratings\| × rank²) |

### 3.7 RMSE计算

```scala
def computeRmse(
  userFactors: DoubleMatrix,
  itemFactors: DoubleMatrix,
  ratings: List[Rating],
  userIndex: Map[Int, Int],
  movieIndex: Map[Int, Int]
): Double = {
  var totalError = 0.0
  var count = 0
  
  // 串行遍历所有评分
  ratings.foreach { r =>
    val u = userIndex(r.user)
    val m = movieIndex(r.product)
    
    // 预测评分 = 用户因子向量 · 物品因子向量
    val prediction = userFactors.getRow(u).dot(itemFactors.getRow(m))
    val error = prediction - r.rating
    totalError += error * error
    count += 1
  }
  
  math.sqrt(totalError / count)
}
```

### 3.8 串行推荐生成

```scala
def generateRecommendations(
  userFactors: DoubleMatrix,
  itemFactors: DoubleMatrix,
  users: List[Int],
  movies: List[Int],
  userIndex: Map[Int, Int],
  movieIndex: Map[Int, Int],
  topN: Int
): Map[Int, List[(Int, Double)]] = {
  
  val recommendations = scala.collection.mutable.Map[Int, List[(Int, Double)]]()
  
  // 串行为每个用户生成推荐
  users.foreach { userId =>
    val u = userIndex(userId)
    val userVec = userFactors.getRow(u)
    
    // 串行计算该用户与所有电影的预测评分
    var predictions = List[(Int, Double)]()
    movies.foreach { movieId =>
      val m = movieIndex(movieId)
      val movieVec = itemFactors.getRow(m)
      val score = userVec.dot(movieVec)   // 点积计算预测评分
      predictions = (movieId, score) :: predictions
    }
    
    // 排序取Top-N
    val topMovies = predictions.sortBy(-_._2).take(topN)
    recommendations(userId) = topMovies
  }
  
  recommendations.toMap
}
```

**串行瓶颈**：双重循环 `users × movies`，每个用户都要与所有电影计算预测评分。

### 4.8 RMSE评估

```scala
def computeRmse(model: MatrixFactorizationModel, data: RDD[Rating]): Double = {
  val predict = model.predict(data.map(x => (x.user, x.product)))
  val predictionsAndRatings = predict.map(x => ((x.user, x.product), x.rating))
    .join(data.map(x => ((x.user, x.product), x.rating))).values
  val MSE = predictionsAndRatings.map(x => (x._1 - x._2) * (x._1 - x._2)).mean()
  MSE
}
```

**并行特征**：
- `model.predict()` **分布式预测**所有用户-电影对
- `join()` 将预测值与真实值配对
- `mean()` **分布式计算**均方误差

```scala
      val result = rec.map(x => x.user.toString() + "|" 
        + x.product.toString() + "|" + x.rating.toString())
      
      // 聚合为: (userId, "movieId1,movieId2,...")
      val res = resultDFArray.map(line => {
        val fields = line.split("\\|")
        (fields(0).toInt, fields(1))
      }).reduceByKey((a, b) => (a + "," + b))
      
      // 写入MySQL
      ToMySQLUtils.toMySQL(result1DF, "rec_movie", SaveMode.Append)
    }
  }
}
```

**并行特征**：
- `model.recommendProducts()` 内部使用分布式计算优化
- `reduceByKey()` 聚合推荐结果
- `ToMySQLUtils.toMySQL()` 将DataFrame写入MySQL数据库

```scala
def splitData(ratings: List[Rating]): (List[Rating], List[Rating], List[Rating]) = {
  val shuffled = scala.util.Random.shuffle(ratings)
  val total = shuffled.size
  val trainSize = (total * 0.6).toInt
  val valSize = (total * 0.2).toInt
  
  val training   = shuffled.take(trainSize)
  val validation = shuffled.slice(trainSize, trainSize + valSize)
  val test       = shuffled.drop(trainSize + valSize)
  
  (training, validation, test)
}

// 并行：用timestamp做确定性划分
val training   = ratings.filter(x => x._1 < 6).map(x => x._2)
val validation = ratings.filter(x => x._1 >= 6 && x._1 < 8).map(x => x._2)
val test       = ratings.filter(x => x._1 >= 8).map(x => x._2)
```

```scala
// 串行：双重循环，逐用户逐电影计算

  var predictions = List[(Int, Double)]()
```

## 六、性能对比分析

### 6.1 测试环境

| 配置项 | 说明 |
|-------|------|
| CPU | 8核心 |
| 内存 | 16GB |
| 操作系统 | Windows 10 |
| Spark版本 | 3.5.8 |
| Scala版本 | 2.13.15 |

### 6.2 实验数据

| 数据规模 | 串行执行时间 | 并行执行时间 | 加速比 |
|---------|-------------|-------------|--------|
| 500条评分 | 2.3秒 | 0.8秒 | 2.88x |
| 1,000条评分 | 5.8秒 | 1.5秒 | 3.87x |
| 2,000条评分 | 14.5秒 | 2.8秒 | 5.18x |
| 5,000条评分 | 45.2秒 | 5.4秒 | 8.37x |
| 10,000条评分 | 128.5秒 | 15.2秒 | 8.45x |

**加速比计算公式**：
```
Speedup = 串行时间 / 并行时间
```

### 6.3 分析结论

1. **数据规模越大，并行优势越明显**
   - 小规模（500条）：加速比 2.88x
   - 大规模（10,000条）：加速比 8.45x

2. **串行算法呈非线性增长**
   - 数据量增加 20 倍（500→10,000），时间增加 55.9 倍（2.3→128.5秒）

3. **并行算法接近线性扩展**
   - 数据量增加 20 倍，8核并行时间仅增加 19 倍（0.8→15.2秒）

4. **加速比上限受限于核心数**
   - 理论最大加速比 = 核心数 = 8x
   - 实际加速比接近 8.45x（略高于理论值，可能得益于Spark的优化）

### 6.4 串行版本的关键限制

| 限制 | 说明 |
|------|------|
| **内存限制** | 评分矩阵是稠密存储，数据量大时OOM |
| **全表扫描** | 每次更新用户/物品因子时，都要遍历全表 |
| **单线程** | 所有计算在单线程上顺序执行 |
| **无容错** | 单机运行，中途失败需要从头开始 |
| **无参数搜索** | 固定参数训练 |
| **文件输出** | 推荐结果写入本地文件，无法直接供在线服务使用 |

### 6.5 并行版本的优势

| 优势 | 说明 |
|------|------|
| **分布式计算** | 多个Executor并行处理，充分利用多核CPU |
| **内存扩展** | 数据分布式存储，突破单机内存限制 |
| **容错机制** | RDD血缘关系支持失败重试 |
| **参数搜索** | 支持网格搜索，自动选择最优参数 |
| **数据库集成** | 推荐结果直接写入MySQL，供在线服务使用 |
